# Artifact Overview

The artifact I chose for my capstone is an Android Inventory Management Application originally created in CS 360 Mobile Architecture and Programming. The app allows users to create accounts, log in, and manage inventory records through a mobile interface. For my capstone, I enhanced this artifact in three categories: software design and engineering, algorithms and data structures, and databases.

## Enhancement One: Software Design and Engineering

This enhancement focused on improving the structure, maintainability, and security of the app. I refactored authentication functionality into an `AuthService` class and inventory operations into an `InventoryService` class. I also improved validation, added password hashing, and strengthened the organization of the application.

[View Software Engineering Narrative](narratives/SoftwareEngineeringNarrative.docx)

## Enhancement Two: Algorithms and Data Structures

This enhancement focused on improving how inventory data is organized, searched, sorted, and retrieved. I added sorting options, filtering features, search functionality, and a `HashMap` data structure to improve inventory item lookup efficiency.

[View Algorithms and Data Structures Narrative](narratives/AlgorithmsDataStructuresNarrative.docx)

## Enhancement Three: Databases

This enhancement focused on improving the database structure of the application. I added new inventory fields, created an inventory history table, and improved the app’s ability to track inventory changes over time.

[View Database Narrative](narratives/DatabaseNarrative.docx)
